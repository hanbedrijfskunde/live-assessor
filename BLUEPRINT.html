A1 · Productoverzicht

Een Nederlandstalige webapplicatie waarmee docenten van HAN Bedrijfskunde het PROMEF Midterm Assessment afnemen en beoordelen. Studenten worden in teams (solo, duo of meer) ingeroosterd op een kalender en beoordeeld op 6 vaste criteria, elk op 4 niveaus. Een AI co-pilot kan gesprekken simuleren of geüploade audio analyseren (via Gemini) om scores en feedback voor te stellen. De docent ziet resultaten in een matrix (cijfer + slaag/zak) en genereert printbare feedbackrapporten. Alle data leeft lokaal in de browser (`localStorage`); import/export verloopt via CSV en JSON.

Kernfunctionaliteit (must-have)

  • Klassen/groepen, teams en studenten beheren (handmatig + CSV-import).
  • Kalenderoverzicht met tijdsloten, pauzes en drag-&-drop herinroostering binnen een groep.
  • Assessmentformulier: per student per criterium een niveau + observatie-tags + vrije notities; duo-score-synchronisatie; per-student reset.
  • AI co-pilot: client-side simulaties (3 scoreprofielen) én Gemini-analyse van audio/transcript.
  • Resultatenmatrix met punten, cijfer en slaag/zak; CSV-export.
  • Printbaar feedbackrapport per student (PDF via browser-print).
  • JSON back-up/herstel en volledige reset.

A3 · Architectuur

export interface TeamAssessment {

A5 · Domeinregels (scoring & status)

Niveau ↔ score

Onder=1, Op=2, Boven=3, Excellent=4. Per student max 6×4 = 24 punten.

Cijfer (getCijferFromScore(total)) — alleen tonen bij volledig (6/6) gescoord

Totaal punten≥24≥20≥16≥14≥12≥10<10
Cijfer10987654

Slaag/zak

Een student is geslaagd ⇔ volledig beoordeeld (alle 6) én geen enkel criterium op niveau "Onder" (score 1). Eén enkele 1 = gezakt, ongeacht het cijfer.

A7 · Schermen (functionele specificatie)

Navigatie via 4 tabbladen in de header: Kalenderoverzicht, Resultatenmatrix, Studenten & Duo's, Dataset & Import. assessment en feedback zijn sub-views (geen tab) bereikt via klik op respectievelijk een teamkaart of een rapportknop.

CalendarOverview (dashboard)

  • Bovenaan 3 statistiektegels: voltooid/totaal + voortgangsbalk; aantal "partial"; totaal teams + #studenten/#klassen.
  • Per groep een kolom met alle tijdsloten (uit generateTimeSlots). Elk slot toont: pauze, teamkaart of lege dropzone.
  • Teamkaart toont teamnummer, statusbadge (Onbeoordeeld/Deels/Voltooid met kleur) en studentnamen. Klik → open assessment.
  • Drag-&-drop: sleep een teamkaart naar een ander slot binnen dezelfde groep; bezet slot ⇒ wissel slottijden (onMoveTeam).
  • Klik op leeg slot ⇒ pauze toevoegen; klik op ✕ in pauze ⇒ verwijderen (onModifyPauze). Pauze toevoegen op bezet slot zet dat team op slotTime=null.

AssessmentView (scoren)

  • Ondersteunt 1..N studenten in een team; isSolo = exact 1. Tabbalk C1–C6 met completion-kleur.
  • Per criterium: info-blok, observatie-tag-pickers per niveau (klik/sleep), en per student een niveau-keuze (4 knoppen) + notitieveld.
  • Tag toevoegen schrijft regel "✓ <tag>" in de notities; bij duo opent een student-picker (solo/duo-direct).
  • Duo-score: checkbox per criterium synchroniseert score+notitie van student 1 naar de rest; vervolgstudenten worden read-only.
  • Per-student reset: knop (↺) in studentkop wist die student volledig ({scores:{},isDuo:{},notes:{}}) na bevestiging; uitgeschakeld als er niets te wissen is.
  • Auto-save bij elke wijziging via onSaveAssessment (herberekent status). Rechterkolom = AiNotulistPanel.

AiNotulistPanel (AI co-pilot)

  • Props: students: {id,name}[] + onApplySuggestions(scores,tags,notes,feedback).
  • Client-side simulatie (geen server): 3 knoppen tonen de echte teamnamen + een scoreprofiel: Alles onvoldoende (alle "Onder"), Random/Voldoende (willekeurig uit Op/Boven/Excellent), Alles uitstekend (alle "Excellent"). buildSimulation() genereert transcript + per-regel competentie-detecties + analyse, gegrond op de rubric-tags. Afspeelloop ~700 ms/regel.
  • Live audio/upload → POST /api/analyze-assessment (Gemini of fallback). Toont transcript + detecties.
  • "Gebruik AI Suggesties" → onApplySuggestions; AssessmentView mapt analyse-keys op volgorde naar de echte studenten.
  • Transcript downloadbaar als .txt.

ResultsView (matrix)

  • Klasselector. Per student: 6 scorecellen (kleur per niveau), totaal punten of "x/6", cijfer-badge, slaag/zak-vlag, knop naar rapport.
  • CSV-export (puntkomma) met scores, totaal, cijfer en status. Legenda met slaag/zak-logica + cijfer-sleutel.

FeedbackView (rapport)

  • Studentselector + printknop (window.print(); print-CSS verbergt .no-print).
  • A4-rapport: kop met kandidaat/datum/klas/assessoren; samenvatting (totaal/24, cijfer, geslaagd/gezakt); per criterium niveau-badge + ✓-tags + vrije tekst; sterke punten/ontwikkelpunten afgeleid uit notities (met fallbacks).

StudentManager

  • 3 formulieren: groep aanmaken (naam/assessoren/datum/slottijd/duur, default pauze 12:00); teams in bulk (verdeelt over sloten, slaat 12:00 over); student toevoegen (groep→team).
  • Overzicht per groep met teams + studenten; verwijderacties (groep cascade-verwijdert teams/studenten/assessments).

DataManager

  • CSV-import (kolommen Student/Naam, Groep/Klas, Team/Teamnummer; delimiter ; of , auto-detect) → reset assessments.
  • JSON back-up maken (alle 4 stukken) + herstellen (valideert aanwezige keys). Voorbeelddata laden. Volledige reset (met bevestiging).

A8 · Server-API

POST /api/simulate-analysis (legacy, optioneel)

Body { scenarioId } → vast scenario-object { title, transcript[], analysis{scores,tags,reasoning,feedback} } of 404. De client gebruikt dit niet meer (simulatie is client-side), maar het endpoint mag blijven bestaan en wordt door de fallback hergebruikt.

POST /api/analyze-assessment (kern)

// Request
{ audioData?: base64, mimeType?: string, transcriptText?: string,
  student1Name: string, student2Name?: string }   // "" => solo

// Response (200)
{ success: true, simulated: boolean,
  transcript: { timestamp, speaker, text }[],
  analysis: {
    scores:    { [naam]: { "1".."6": 1..4 } },
    tags:      { [naam]: { "1".."6": string[] } },
    reasoning: { [naam]: { "1".."6": string } },
    quotes:    { [naam]: { "1".."6": string } },
    feedback:  { [naam]: { strengths:string[], improvements:string[] } }
  } }
// Fout: 500 { error, details }

Gedrag: zónder geldige GEMINI_API_KEY → graceful fallback met een vast scenario (namen worden geremapt naar student1Name/student2Name). Mét sleutel → Gemini generateContent met responseSchema (objectsleutels exact gelijk aan de aangeleverde studentnamen; 1 student bij solo), temperature 0.2, Nederlandstalige systeeminstructie.
